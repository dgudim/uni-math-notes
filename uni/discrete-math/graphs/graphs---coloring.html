Basic information

Definition

Graph coloring is a special case of graph labeling

  • Assignment of labels traditionally called "colors" to elements of a graph
  • Coloring the vertices of a graph such that no two adjacent vertices are of the same color (Vertex coloring)
  • Coloring the edges such that no two adjacent edges are of the same color
  • Total coloring of vertices and edges
  • Face coloring of a planar graph assigns a color to each face or region so that no two faces sharing a boundary have the same color

Chromatic polynomial and number

Definition

Chromatic polynomial CG(λ) shows in how many different ways we can color vertices using λ colors

Definition

Chromatic number - Smallest possible amount of colors needed to color vertices
It is equal to smallest integer with which chromatic polynomial GG(n)≠0

Let's have λ colors. For this graph there will be λ(λ−1)(λ−2)(λ−2)=CG(λ) ways to do it

  • λ for the first vertex, λ−1 for second vertex and λ−2 for top and bottom vertices
Note

CG(3)≠0
CG(2)=0
CG(1)=0
So, minimal amount of colors (chromatic number) is 3

Some formulas for known graph types

Theorem

If G=G1∪G2∪⋯∪Gn where Gi - it's connected components, then CG(λ)=CG1(λ)∗CG2(λ)∗⋯∗CGn(λ)

Tldr

If graph G can be colored using K colors, then some connected component of it can be colored using K colors

CGe^=CG(λ)+CG/e(λ)

Heuristics

Greedy Algorithm (FF)

  1. Assign the first color to the first vertex
  2. The remaining vertices are assigned the smallest possible color
    • A color is not available if the neighboring vertex already has that color

Colors:

  1. #c/red red
  2. #c/green green
  3. #c/blue blue
  4. gray

Vertices: 1 2 3 4 5

  • 1 vertex: #c/red red
  • 2 vertex: #c/red red
  • 3 vertex:
    • ! #c/red red (impossible)
    • #c/green green
  • 4 vertex: #c/red red
  • 5 vertex:
    • ! #c/red red (impossible)
    • ! #c/green green (impossible)
    • #c/blue blue

Random Queue (RS)

  1. List vertices randomly
  2. Apply greedy algorithm

  • 2 vertex: #c/red red

Largest First (LF)

Vertices with a smaller degree usually have a greater choice of possible colors

  1. List vertices in descending order of degrees (vertices with equal degrees are taken from list as is (with the smallest number))
  2. Apply greedy algorithm

  1. #c/red red
  2. #c/green green
  3. #c/blue blue
  4. gray

Vertices: 3 5 1 4 2

  • 3 vertex: #c/red red
  • 5 vertex:

    • #c/green green
  • 1 vertex:
    • ! #c/red red (impossible)
    • ! #c/green green (impossible)
    • #c/blue blue
  • 4 vertex:
    • ! #c/red red (impossible)
    • ! #c/green green (impossible)
    • #c/blue blue
  • 2 vertex:
    • ! #c/red red (impossible)
    • #c/green green

Smallest Last (SL)

  • h-degree: the number of colors not available for this vertex
  • k-degree: the number of unpainted vertices adjacent to the vertex

Sorting rules:

  • Vertex with the biggest h-degree is preferred
  • If h-degrees of several vertices are the same, take one with the biggest k-degree
  • If k-degrees of several vertices are the same, take one with the smallest number (closest to the start of the list)

Colors:

  1. #c/red red
  2. #c/green green
  3. #c/blue blue
  4. gray
  • 3 vertex (biggest k-degree, 4): #c/red red
  • 5 vertex (biggest k-degree, 2):
    • ! #c/red red (impossible)
    • #c/green green
  • 1 vertex (smaller number):
    • ! #c/red red (impossible)
    • ! #c/green green (impossible)
    • #c/blue blue
  • 4 vertex (biggest h-degree, 4):
    • ! #c/red red (impossible)
    • ! #c/green green (impossible)
    • #c/blue blue
  • 2 vertex:
